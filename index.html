# 3. Convertir columnas clave a numéricas (si están como texto)

# Función para limpiar columnas numéricas con problemas
limpiar_numero <- function(x) {
  x <- gsub(",", ".", x)      # Reemplaza coma por punto (para decimales)
  x <- gsub("%", "", x)       # Quita porcentaje si lo hubiera
  x <- gsub("-", "0", x)      # Reemplaza guiones por 0
  x <- gsub("[^0-9\\.]", "", x) # Quita cualquier cosa que no sea número o punto
  as.numeric(x)               # Convierte a número
}

# Aplicar limpieza a columnas clave

Data <- Call.Center.Data %>%
  mutate(
    waiting_time_avg = limpiar_numero(waiting_time_avg),
    answer_speed_avg = limpiar_numero(answer_speed_avg),
    service_level_20_seconds = limpiar_numero(service_level_20_seconds) / 100,  # Si viene en porcentaje
    answer_rate = limpiar_numero(answer_rate) / 100
  )

📝 Resumen Ejecutivo

Este reporte analiza metricas clave de desempeño en un contact center, donde se intenta evaluar:

-Nivel de servicio

-Tiempo de espera

-Llamadas abandonadas

-Relación entre carga de llamadas y atención

Conclusiones y Recomendaciones

🔹 Desempeño Aceptable pero no Óptimo

La mayoría de los indicadores muestran un rendimiento razonable. Sin embargo, el análisis del nivel de servicio indica que, aunque gran parte de los registros está por encima del 70%, aún existe una proporción significativa por debajo del 50%, lo que sugiere inconsistencias operativas.

🔹 Problemas de Capacidad de Respuesta

Se evidenció que no todas las llamadas entrantes están siendo respondidas.

🔹 Impacto del Tiempo de Espera en la Satisfacción del Cliente

El tiempo promedio de espera está claramente relacionado con el abandono de llamadas. Mientras mayor sea la espera, más usuarios cuelgan antes de ser atendidos, lo cual representa una pérdida directa de oportunidad de atención.

🔹 Limitaciones del Análisis

La ausencia de variables temporales (como fecha, hora o turno) y de identificación del agente limita el análisis en profundidad. No se puede determinar en qué momentos ni con qué personal se presentan los mayores problemas, lo que restringe la toma de decisiones más dirigidas.

🔹 Recomendaciones

  • Identificar registros con nivel de servicio < 0.5 y evaluar sus características particulares.
  • Implementar sistemas de monitoreo en tiempo real para ajustar la capacidad de respuesta en momentos críticos.
  • Considerar una recolección de datos más detallada, incluyendo hora, fecha, agente y canal de contacto, para futuros análisis más precisos.
