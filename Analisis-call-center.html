# 3. Convertir columnas clave a numricas (si estn como texto)

# Funcin para limpiar columnas numricas con problemas
limpiar_numero <- function(x) {
  x <- gsub(",", ".", x)      # Reemplaza coma por punto (para decimales)
  x <- gsub("%", "", x)       # Quita porcentaje si lo hubiera
  x <- gsub("-", "0", x)      # Reemplaza guiones por 0
  x <- gsub("[^0-9\\.]", "", x) # Quita cualquier cosa que no sea nmero o punto
  as.numeric(x)               # Convierte a nmero
}

# Aplicar limpieza a columnas clave

Data <- Call.Center.Data %>%
  mutate(
    waiting_time_avg = limpiar_numero(waiting_time_avg),
    answer_speed_avg = limpiar_numero(answer_speed_avg),
    service_level_20_seconds = limpiar_numero(service_level_20_seconds) / 100,  # Si viene en porcentaje
    answer_rate = limpiar_numero(answer_rate) / 100
  )

Resumen Ejecutivo

Este reporte analiza metricas clave de desempeo en un contact center, donde se intenta evaluar:

-Nivel de servicio

-Tiempo de espera

-Llamadas abandonadas

Conclusiones y Recomendaciones

Desempeo Aceptable pero no ptimo

La mayora de los indicadores muestran un rendimiento razonable. Sin embargo, el anlisis del nivel de servicio indica que, aunque gran parte de los registros est por encima del 70%, an existe una proporcin significativa por debajo del 50%, lo que sugiere inconsistencias operativas.

Problemas de Capacidad de Respuesta

Se evidenci que no todas las llamadas entrantes estn siendo respondidas.

Impacto del Tiempo de Espera en la Satisfaccin del Cliente

El tiempo promedio de espera est claramente relacionado con el abandono de llamadas. Mientras mayor sea la espera, ms usuarios cuelgan antes de ser atendidos, lo cual representa una prdida directa de oportunidad de atencin.

Limitaciones del Anlisis

La ausencia de variables temporales (como fecha, hora o turno) y de identificacin del agente limita el anlisis en profundidad. No se puede determinar en qu momentos ni con qu personal se presentan los mayores problemas, lo que restringe la toma de decisiones ms dirigidas.

Recomendaciones

  • Identificar registros con nivel de servicio < 0.5 y evaluar sus caractersticas particulares.
  • Implementar sistemas de monitoreo en tiempo real para ajustar la capacidad de respuesta en momentos crticos.
  • Considerar una recoleccin de datos ms detallada, incluyendo hora, fecha, agente y canal de contacto, para futuros anlisis ms precisos.
